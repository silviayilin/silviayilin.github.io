Lazy loading 延遲載入

發表於 2021-12-14 更新於 2021-12-16 分類於 Angular
前言

目前工作是在銀行專案，初期尚未導入延遲載入，
隨著開發的模組(NgModule)達到千單位，初始載入時間至少超過一分鐘，
團隊透過延遲載入结合路由配置，在需要時才動態加載相應模組以解決網站開啟太慢的問題。

實作

建立專案

建立branch延遲載入路由模組

宣告的路由 branch 新增到以 --module 選項指定的模組中宣告的 routes 陣列中。

使用 Chrome 開發者工具，將初始載入的資訊先清除
測試點擊 ```Branch``` 來確認一下branch module真的是延遲載入的。

SPA（Single Page Application）

發表於 2021-12-14 更新於 2021-12-15 分類於 Angular

單頁面應用的所有應用程式功能都存在於同一個 HTML 頁面中，相較傳統的客戶端(Client)一個請求從伺服器端(Server)重新載入整個新頁面，SPA不需跳轉重新載入頁面就可以支援客戶端訪問應用的各種特性。

SPA初次載入(一次性載入所有元件)應用程式相關代碼與呈現視圖會需較多時間，意味著包含不是初始必要的函式庫與模組，而延遲載入的方式可根據使用者請求才載入某些特定模組，以提升初始載入效能。
